# SCRIPT — Termada Mission Control Build Week

**Voice:** Samantha (macOS English)
**Voice settings:** 168 words per minute
**Voice direction:** Calm, credible, technically precise, with deliberate emphasis on approval and proof.

---

## Line 1 — The trust gap (Frame 1)

**Time:** 0 – 16s
**Delivery:** Open directly and land “trustworthy outcome.”

    AI agents can run shell commands. But when the target is a real environment, a successful command is not the same as a trustworthy outcome. This is Termada Mission Control: persistent, policy-controlled missions for AI agents operating local and remote systems.

## Line 2 — Build Week delta (Frame 2)

**Time:** 16 – 40s
**Delivery:** Clearly separate the existing foundation from the new extension.

    Before Build Week, Termada already provided persistent terminal sessions, SSH, policy rules, human approvals, secrets redaction, and a tamper-evident audit. During Build Week, we added the mission layer: durable goals and plans, interruption recovery, runtime-verified steps, an evidence-first dashboard, and signed-by-hash reports.

## Line 3 — Diagnosis (Frame 3)

**Time:** 40 – 70s
**Delivery:** Faster, with emphasis on real observations.

    Here, GPT-5.6 in Codex receives one goal: restore a broken checkout health endpoint. It creates a dedicated mission and turns the goal into four testable steps. The first probe observes a real HTTP 503. The next command identifies the faulty service mode. Both steps are marked passed only because Termada observed mission-owned jobs exit with code zero.

## Line 4 — Approval (Frame 4)

**Time:** 70 – 100s
**Delivery:** Slow down at the policy boundary and “Allow once.”

    Now the agent proposes the protected repair. Termada's policy engine stops the command. The dashboard shows the exact command and the matching approval rule. Nothing continues until a human chooses Allow once. After approval, the same persistent session applies the fix.

## Line 5 — Verified outcome (Frame 5)

**Time:** 100 – 124s
**Delivery:** Confident, without hype.

    A final probe observes a real HTTP 200 with status ok. The mission reaches four of four verified steps and records a clear outcome. The final evidence report separates agent notes from runtime facts. It includes every correlated command, the human approval, audit-chain sequence and record hashes, and its own SHA-256.

## Line 6 — Roles and CTA (Frame 6)

**Time:** 124 – 141s
**Delivery:** Distill the model/runtime split, then finish cleanly.

    GPT-5.6 supplies adaptive planning and diagnosis. Termada supplies deterministic execution, policy, persistence, ownership, and proof constraints. The model can propose success. The runtime decides whether the evidence supports it. Termada is open source and local first. Let the agent move fast, without losing control of what actually happened.
