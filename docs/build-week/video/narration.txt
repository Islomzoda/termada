AI agents can run shell commands. But when the target is a real environment, a successful command is not the same as a trustworthy outcome.

This is Termada Mission Control: persistent, policy-controlled missions for AI agents operating local and remote systems.

Before Build Week, Termada already provided persistent terminal sessions, SSH, policy rules, human approvals, secrets redaction, and a tamper-evident audit. During Build Week, we added the mission layer: durable goals and plans, interruption recovery, runtime-verified steps, an evidence-first dashboard, and signed-by-hash reports.

Here, GPT-5.6 in Codex receives one goal: restore a broken checkout health endpoint. It creates a dedicated mission and turns the goal into four testable steps.

The first probe observes a real HTTP 503. The next command identifies the faulty service mode. Both steps are marked passed only because Termada observed mission-owned jobs exit with code zero.

Now the agent proposes the protected repair. Termada's policy engine stops the command. The dashboard shows the exact command and the matching approval rule. Nothing continues until a human chooses Allow once.

After approval, the same persistent session applies the fix. A final probe observes a real HTTP 200 with status ok. The mission reaches four of four verified steps and records a clear outcome.

The final evidence report separates agent notes from runtime facts. It includes every correlated command, the human approval, audit-chain sequence and record hashes, and its own SHA-256.

GPT-5.6 supplies adaptive planning and diagnosis. Termada supplies deterministic execution, policy, persistence, ownership, and proof constraints. The model can propose success. The runtime decides whether the evidence supports it.

Termada Mission Control gives developers and small operations teams a practical trust layer between an AI agent and real infrastructure: inspectable while it runs, interruptible by a human, recoverable after failure, and reviewable when the work is done.

Termada is open source, local first, and available for macOS and Linux. Let the agent move fast, without losing control of what actually happened.
